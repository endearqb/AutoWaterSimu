# 任务计划 (Task Plan): WaterTAP 风格计算器 HTML 中文预览页

## 目标 (Goal)
实现一个独立的静态 HTML 中文预览页，用于验证 5 个 WaterTAP 风格纯前端快算器的内容结构、公式交互和说明方式。页面需要可直接通过前端静态资源访问，并为后续迁移到 React/TypeScript 做好计算逻辑与信息结构准备。

## 关键问题 (Key Questions)
1. 如何让静态页既方便直接预览，又能最大程度复用到后续 React 集成？
2. 如何用透明、可解释的工程快算公式表达 WaterTAP 的计算主题，而不是假装复刻求解器？
3. 如何在单页内清晰呈现输入、结果、docs、边界和 WaterTAP 来源映射？

## 决策记录 (Decisions Made)
| 决策 | 理由 |
|----------|-----------|
| 预览页放在 `frontend/public/previews/` 下 | 启动前端后可直接通过 URL 访问，不需要先接 React 路由 |
| 所有计算逻辑写成独立 JS 纯函数 | 便于后续迁移到 TS 或抽成共享公式层 |
| 每个计算器提供 2 组示例按钮 | 便于快速验证常规工况和边界工况 |
| 页面重点放在内容验证而非最终首页视觉 | 当前目标是验证计算结构与说明方式，避免过早耦合首页设计 |

## 备注 (Notes)
- 本页默认是“工程快算 + 概念对齐”，不承诺复刻 WaterTAP/Pyomo 的严格求解结果。
- 页面应明确标注使用假设、适用场景和不适用场景，避免误导为正式设计工具。
