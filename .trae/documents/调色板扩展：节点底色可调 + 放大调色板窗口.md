## 目标
- 让“主题调色板”同时影响节点的玻璃底色（填充）而不仅是字体/边框/把手，让画布节点的整体色调随主题变化。
- 放大调色板弹层窗口，增加更多色块以及预览分配效果，提升可操作性。

## 现状
- 节点玻璃底色读取 CSS 变量：`GLASS_BG[tint]`（frontend/src/components/Flow/nodes/utils/glass.ts:28–38），目前只定义了默认变量但没有在调色板应用中覆盖。
- 我们已将 accent/outline/handle 颜色改为 CSS 变量读取（glass.ts:40–74），并在 `useThemePaletteStore.applyPalette` 写入对应变量，但尚未写入 `--chakra-colors-glass-tint-<tint>`。
- 调色板弹层尺寸约为 `w=280px, h=180px`，色块数量较少（SimulationActionPlate.tsx:985–1006）。

## 变更点
- 覆盖节点玻璃底色：在 `useThemePaletteStore.applyPalette` 中，除了设置 `--chakra-colors-glass-accent-* / outline-* / handle-*`，同时写入 `--chakra-colors-glass-tint-<tint>`，用于 `getGlassNodeStyles` 的背景色（frontend/src/components/Flow/nodes/utils/glass.ts:121）。
- 颜色生成策略：将调色板颜色经过“适度变浅”并加入透明度，用于玻璃底色，以保证可读与通透性（推荐 alpha≈0.45）。
- 调色板窗口放大：将弹层根容器尺寸调整到 `w≈420px, h≈260px`；色块增至 24 个；增加“预览行”展示每个 tint 的最终赋色，便于确认。

## 技术实现
- 在 `frontend/src/stores/themePaletteStore.ts`：
  1. 扩展 `applyPalette(order)`：为每个 tint 除了写入 `accent/outline/handle`，还写入 `--chakra-colors-glass-tint-<tint>`。
  2. 玻璃底色值生成：
     - 基于当前 `palette[idx]`（HEX）做一次与白色混合（参考 `mixWithWhite`：frontend/src/components/DataAnalysis/ColumnVisualization/utils/colorSchemes.ts:218–229），mix 比例约 0.35–0.50；
     - 添加透明度（使用 8 位 HEX：`#RRGGBBAA`，AA≈`73` 对应 0.45），或直接生成 `hsla(r,g,b,0.45)`；
     - 写入 `document.documentElement.style.setProperty('--chakra-colors-glass-tint-<tint>', value)`。
  3. 保留 `reset()` 恢复默认变量（包括移除 `--chakra-colors-glass-tint-*`）。
- 在 `frontend/src/components/Flow/inspectorbar/SimulationActionPlate.tsx`：
  1. 放大弹层：将主题弹层容器改为 `w="420px" h="260px"`；
  2. 色块扩展：`EXTENDED_COLOR_PALETTE.slice(0, 24)`；
  3. 预览行：新增一行小方块按顺序展示 `order` 映射后的颜色（带 tint 标签：`input/output/asm1/asm3/asmslim/default/...`），方便确认；
  4. 保持现有按钮交互和关闭逻辑不变。

## Tint 分配与兼容
- ASM1 页面：优先 `['asm1','input','output','default','asm3','asmslim']`。
- ASM3 页面：优先 `['asm3','input','output','default','asm1','asmslim']`。
- ASM1Slim 页面：优先 `['asmslim','input','output','default','asm1','asm3']`。
- 物料平衡：`['input','output','default','asm1','asm3','asmslim']`。
- 不足色长度循环取用；多余 tint 用队列继续分配。

## 验证
- 类型检查：`cd frontend; npx tsc --noEmit`。
- 运行前端：`npm run dev`，分别在 4 个页面测试：
  - 应用主题后节点底色（玻璃背景）、边框、把手、MiniMap 节点颜色联动变化；
  - 预览行颜色与实际节点颜色一致；
  - 重置后恢复默认玻璃色。
- 可用 `isWCAGCompliant`（colorSchemes.ts:669–673）抽检对比度，避免极低对比主题。
