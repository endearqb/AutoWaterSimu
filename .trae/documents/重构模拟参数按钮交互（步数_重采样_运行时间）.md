## 修改目标
- 将“每小时步数”“重采样间隔”“运行时间”改为与“小地图/分析结果”一致的按钮交互：默认显示当前值，鼠标悬停时显示中文标签。
- 保留中间的“开始”按钮，左侧放“每小时步数/重采样间隔”，右侧新增“运行时间”按钮。
- 点击“运行时间”按钮，参考“每小时步数”按钮的效果，在按钮上方弹出带动画的 Slider。

## 具体实现
1. 布局调整
- 在中部 `HStack` 中：左侧保留两个按钮（步数、重采样），右侧新增一个按钮（运行时间），中间保持“开始”按钮。

2. 统一按钮样式与悬停文案
- 三个参数按钮统一使用当前已有的 `getToggleButtonStyles(...)` 与内部 `HStack` 结构（图标 + 文案），通过本地 `showXXXLabel` 状态控制文案伸缩。
- 默认显示具体值；`onMouseEnter/Leave` 切换为中文标签。
  - 步数：默认 `${simulationSteps}步`，悬停“每小时步数”。
  - 重采样：默认 `${(1 / samplingIntervalHours).toFixed(1)}h-1`，悬停“重采样间隔”。
  - 运行时间：默认 `${Math.min(simulationHours, maxSimulationTime)}h`，悬停“运行时间”。

3. 弹层交互
- 步数、重采样：沿用现有弧形按钮弹层（`computeArcPositions` + `Portal`）与动画 `fadeInUp/fadeOutUp`，不改动选项与样式，点击后分别调用：
  - `validateAndUpdateParameters({ steps_per_hour: value })`
  - `validateAndUpdateParameters({ sampling_interval_hours: value })`
- 运行时间：新增 `isHoursOpen/isHoursClosing/hoursBtnRef/showHoursLabel`；点击按钮开关弹层，在按钮上方展示 `Slider.Root`：
  - `value={[Math.min(simulationHours, maxSimulationTime)]}`
  - `onValueChange={(d) => validateAndUpdateParameters({ hours: d.value[0] })}`
  - `min={config.hours.min}` `max={maxSimulationTime}` `step={config.hours.step}`
  - 复用 `sliderTrackProps` 与 `renderSliderThumb(isHoursDisabled)`；动画与定位复用 `panelRect` 与 `getAnchorCenter(...)`。

4. 移除重复的运行时间 Slider
- 隐藏或删除“高级参数”区块中的运行时间 Slider，避免重复展示与视觉冲突；保留区块其他内容不变。

5. 禁用与样式一致性
- 保持 `isStartDisabled`、`isHoursDisabled` 对按钮与弹层的禁用逻辑（半透明、不可点击）。
- 保持 `accentColor`、`accentOutlineColor`、`getBubbleButtonStyles` 现有视觉风格与动画。

## 验证步骤
- 运行类型检查：`cd frontend; npx tsc --noEmit`。
- 本地预览：`cd frontend; npm run dev`。
- 交互验收：
  - 三个按钮默认显示值，悬停显示中文标签，过渡顺滑。
  - 点击“步数/重采样”显示弧形选项并可更新；点击“运行时间”向上弹出 Slider 并可更新。
  - 禁用态不可操作、样式半透明；弹层定位与遮挡正确。

## 注意事项
- 计算与文案：重采样默认值展示为频率 `${(1 / samplingIntervalHours).toFixed(1)}h-1`；时间展示 `xh`；步数展示 `x步`。
- 不更改 `useSimulationController` 的参数与校验逻辑，仅复用其 `validateAndUpdateParameters` 与边界 `maxSimulationTime`。
- 保持现有阴影、圆角、颜色 token 与动画一致，避免主题偏差。
