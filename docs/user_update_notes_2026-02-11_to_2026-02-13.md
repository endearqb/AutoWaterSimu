# AutoWaterSimu 普通用户更新说明（2026-02-11 ～ 2026-02-13）

## 1. 这三天我们为你带来了什么

在这三天里，AutoWaterSimu 从“可用的仿真工具”进一步升级为“可复用、可扩展、可协同”的建模平台，重点完成了三件事：

1. 让用户自定义模型（UDM）从“能建”变成“好建、好管、好复用”。
2. 让流程图仿真支持“分时段参数输入”，更贴近真实工况切换。
3. 让一张流程图开始支持“多 UDM 模型混合运行（Hybrid）”，并增加提交前校验，减少无效计算。

如果你是工艺工程师、运营人员、方案咨询人员，最直接的变化是：  
建模更快、改模更安全、结果更可信、跨场景复用更容易。

## 2. 按天看更新（面向普通用户）

## 2026-02-11：UDM 建模能力成体系上线

### 你能直接用到的新功能

1. 新增 UDM 模型库页面（`/_layout/udmModels`）。
2. 新增 UDM 模型编辑器页面（`/_layout/udmModelEditor`）。
3. 支持模板建模、模型复制、发布状态切换、删除、查询。
4. 支持 Petersen 矩阵编辑、表达式校验、参数范围维护。
5. 在 UDM 流程图内可直接弹窗打开模型编辑器，无需跳转。

### 你会感受到的体验改进

1. UDM 计量关系支持表达式（`stoich_expr`），不再只能写死数值。
2. 计算参数面板改成“按模型快照动态加载”，参数不再硬编码。
3. 模型参数总览可按节点聚合真实参数与组分，排查更直观。
4. 编辑器做了多项易用性增强：粘性列、批量粘贴、校验定位、未保存离开提醒。

### 这一日的核心价值

UDM 从“高级功能”变成了“日常可操作能力”，用户可以真正沉淀自己的模型资产库，而不是一次性试验。

---

## 2026-02-12：编辑体验升级 + 分时段输入能力成型

### 你能直接用到的新功能

1. UDM 编辑器中的 `rateExpr` 与 `stoich` 支持弹窗编辑。
2. 弹窗右侧支持变量/参数快捷插入，按光标位置写入。
3. UDM 组件新增 `freeze`（冻结）开关，可指定某组分 `dC/dt=0`。
4. 仿真引入“时段计划”编辑器：支持新增、复制、排序、删除时段，并为每条边配置 `flow/param_a/param_b` 分段覆盖。
5. 图表支持“时段分割线 / 参数变更注记”开关，便于回看每次工况切换的影响。

### 你会感受到的体验改进

1. 表格改为更适合大模型编辑的布局（最小列宽、横向滚动、关键列冻结）。
2. 表达式提示改成“实时高亮 + 实时 warning”，减少提交后返工。
3. 多语言（中/英）文案统一，提示更一致，跨团队沟通成本更低。
4. 提交前增加时段合法性校验，不合法直接阻断，避免浪费算力。

### 这一日的核心价值

系统开始支持“分时段工况”的标准化建模，为真实运行中常见的分班、切换、冲击负荷等场景做好了基础设施。

---

## 2026-02-13：Hybrid UDM（多模型同图）一期落地

### 你能直接用到的新功能

1. 右侧“连接参数设置”新增“时间分段设置”入口，可直接在边参数面板编辑时段与分段参数。
2. 首次进入且无时段时，自动生成默认时段 `[0, hours]`，开箱即用。
3. 新增 Hybrid Setup 对话框：可一次选择多个 UDM 模型，并配置模型对映射关系。
4. UDM 节点属性新增“Hybrid 模型绑定”下拉，每个节点可绑定不同 UDM 模型。
5. 新增提交前 Hybrid 校验接口（`/api/v1/udm/validate-hybrid-flowchart`），在任务提交前先检查映射完整性和可计算性。

### 你会感受到的体验改进

1. 未覆盖的分段参数默认继承，且界面直接显示继承数值，减少“隐式状态”误解。
2. 时段编辑布局更适合长流程图：面板扩展、横向滚动、顶部固定。
3. Hybrid 相关前端文案完成国际化，中文/英文用户体验一致。

### 这一日的核心价值

从“一张图一个模型”走向“一张图多模型协同”，是项目迈向异构工艺混合仿真的关键一步。

## 3. 这些更新对项目、业务、行业意味着什么

## 对项目本身的意义

1. 产品能力从“单次仿真工具”升级为“可沉淀模型资产的平台”。
2. 前后端契约更完整（校验接口、统一字段、OpenAPI 同步），迭代风险下降。
3. UDM 引擎支持局部变量到全局变量投影，为混合模型扩展打下技术底座。

## 对业务落地的意义

1. 建模交付速度更快：模板、复制、弹窗编辑、快捷插入显著降低建模时间。
2. 结果可信度更高：提交前校验、映射完整性检查减少“算了才发现错”的情况。
3. 场景覆盖更广：分时段输入与多模型同图支持真实生产中的工况变化与异构单元。
4. 知识复用更强：企业可持续积累自有 UDM 库，形成内部方法论与标准件。

## 对行业发展的意义

1. 推动水处理数字化从“固定模型仿真”走向“可编排的模型工程”。
2. 为复杂工艺线提供更可执行的数字孪生路径：分时段 + 混合模型 + 可解释校验。
3. 降低先进建模门槛，让更多一线工程团队可以使用而非依赖少数专家手工维护。

## 4. 功能上线后的使用场景畅想

## 场景 A：市政污水厂分时段运行优化

白天/夜间进水差异大时，可用时段计划配置不同边参数，叠加图表分割线快速对比每个时段的响应，辅助运行策略优化。

## 场景 B：工业园区多来源废水协同处理

不同处理单元可绑定不同 UDM 模型，在同一流程图中混合仿真；通过模型对映射处理变量交互，提升跨单元联调效率。

## 场景 C：咨询与方案比选

咨询团队可复用模板模型，按项目快速复制和微调参数，在更短时间内产出多方案对比与解释性报告。

## 场景 D：企业内部模型资产化

企业将历史项目经验沉淀为 UDM 模型库，新项目可直接调用、组合、迭代，逐步建立“模型即资产”的能力。

## 场景 E：教学与培训

通过可视化流程图 + 实时表达式提示 + 结果分析面板，帮助新人理解参数、反应关系与工况变化的因果链条。

## 5. 给普通用户的建议用法（从今天开始）

1. 先在 UDM 模型库选模板复制出你的项目模型，再做小步修改并保存版本。
2. 在流程图中先用单模型跑通，再开启 Hybrid，逐节点绑定模型。
3. 对存在工况切换的项目，优先开启时段计划，不要把所有参数平均化。
4. 提交前关注校验反馈，先消除映射与配置错误，再启动长时仿真。
